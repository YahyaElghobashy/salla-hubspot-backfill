1 · Credentials

The HubSpot private-app token needs CRM read/write scopes (contacts, orders/commerce, line items, custom objects). The relay secret is the random string inside your Make relay scenario's filters.

2 · Salla relay

Import relay/relay_blueprint.template.json into Make (replace the secret placeholder and select your Salla connection on both modules), turn it ON, and paste the webhook URL here. The test fetches your store's order-status list through it.

3 · Google Sheets & Drive (optional)

Live audit rows and order-JSON archive. Skip it and the engine keeps complete local CSV mirrors instead. To enable: create an OAuth Desktop client in Google Cloud Console (Sheets + Drive APIs), download it as credentials.json next to serve.py.

4 · Pipeline mapping

Pick the order pipeline stage used at creation, then map each Salla status to a stage (statuses come from your store via the relay test; stages from your portal). Unmapped statuses fall back to the default stage.

5 · Backfill window & save

The engine sweeps this window in fixed slots, oldest first, and keeps its position in cursor.json. Stop and resume any time; deduplication makes rescans free.

Advanced — concurrency & adaptive pacing (safe defaults)

Lanes share the pacing globally, so N lanes never exceed one lane's budget. If other integrations search the same portal, lower the search limit to your fair share of the 5/s account pool.

Run control

Dry run reads everything and writes nothing — it prints a would-create plan per order. Live runs sit behind the same RUN confirmation the terminal uses. Stop is always graceful: the engine finishes the current order, the cursor stays safe, resume is free.

Lanes process different orders concurrently while sharing the same global rate pacing. 3–4 is the useful range — beyond that the account-wide HubSpot search ceiling is the wall and extra lanes add only 429 risk.

Recommended first flight

1) Dry run with max orders 2 — review the plan in Logs. 2) Live with max orders 2 — verify both orders in HubSpot (name, stage, contact, line items). 3) Live, no cap. This mirrors the tested production rollout of this engine.
